annie.dataset.sources¶

Generic, extensible data-source registry (domain).

Annie is dataset-agnostic: a dataset is described as an ordered list of data sources the user adds on the Dataset tab, rather than four fixed folders. Each source is one of:

  • SourceKind.VIDEO — a folder of source videos (the mandatory spine; one Browse row per video). Singleton.

  • SourceKind.VDET — a folder of raw-detection .vdet files. Singleton.

  • SourceKind.TRACK — a folder of __track{N}.csv files. Singleton.

  • SourceKind.CSV — a label/metadata CSV joined to videos by a chosen key column. A CSV plays one of two roles (CsvRole): labels (its selected value columns become Browse tags/filters; many allowed) or main_character (one value column is the active-track id; singleton).

This module is pure domain — no NiceGUI, no config, no torch. The registry is the single source of truth the scanner (annie.scanning) consumes.

class annie.dataset.sources.SourceRegistry[source]¶

An ordered set of data sources with singleton rules.

Folder kinds (video/vdet/track) and the main-character CSV are singletons: adding one replaces any existing source of that role. Label CSVs are unlimited, keyed by path (re-adding the same file replaces it).
