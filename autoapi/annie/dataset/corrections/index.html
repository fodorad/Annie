annie.dataset.corrections¶

Protagonist correction service.

This is the service surface the UI uses to read and fix which track is the active “protagonist” of a video. It composes the domain-level participant resolver (annie.parsers.participants) and the geometric hit-test (annie.color) so the UI never reaches past the service layer.

The correction flow: the annotator clicks the true protagonist on one of the five highlighted frames; hit_test_frame() maps the click to a track_id; set_active_track() persists it (manual _manual sibling, upsert by uuid); the frames are re-rendered so the new active track shows green.
