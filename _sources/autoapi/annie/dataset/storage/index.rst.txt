annie.dataset.storage
=====================

.. py:module:: annie.dataset.storage

.. autoapi-nested-parse::

   SQLite-backed review-status store (Browse tab curation state).

   This holds *curation* state per video — a good/bad verdict, an optional note, and
   an "add to annotator" flag — not fine-grained annotation content (that lives in
   the protagonist ``_manual`` CSV and, later, per-video JSON). The store is a real
   file (default ``~/.annie/annie.db``) so it survives restarts, and is keyed by
   :attr:`annie.models.VideoEntry.key`.

   Every video is **liked (good) by default**: a row only exists once the user
   interacts, and the UI treats "no row" as good. The store is exportable to CSV/JSON
   in one call and importable via upsert, so a reviewer's curation travels with them.

.. py:data:: Verdict

   A review verdict. ``None`` (no row) is treated as ``"good"`` by the UI.

.. py:class:: ReviewRecord

   One persisted review row.

   .. attribute:: row_key

      Stable per-row identity (:attr:`annie.models.VideoEntry.key`).

   .. attribute:: video_id

      The video this review belongs to.

   .. attribute:: annotation_suffix

      The annotation suffix, or ``None`` for an exact match.

      ``"good"``, ``"bad"``, or ``None`` if untouched (treated as good).

      Whether the video is queued for the Annotator tab.

   .. attribute:: updated_at

      ISO-8601 UTC timestamp of the last change.


.. py:class:: ReviewStore(db_path: str | pathlib.Path)

   A thin, connection-per-call wrapper over the ``review`` table.

   The store opens a fresh connection for each operation, which keeps it safe to
   call from the render worker threads and the UI thread alike without sharing a
   connection across threads.

   Open (creating if needed) the review database at ``db_path``.

   :param db_path: Filesystem path to the SQLite file. Parent directories are
                   created as needed.

   .. py:method:: upsert(row_key: str, video_id: str, annotation_suffix: str | None, *, verdict: Verdict | None = None, note: str | None = None, annotate: bool | None = None) -> ReviewRecord

      Insert or update a review row, preserving fields left as ``None``.

      Passing ``verdict=None`` / ``note=None`` / ``annotate=None`` keeps any
      existing value, so the good/bad toggle, the note, and the annotator flag
      update independently.

      :param row_key: Stable row identity (primary key).
      :param video_id: The video id, stored for export/grouping.
      :param annotation_suffix: The annotation suffix, or ``None``.
      :param verdict: New verdict, or ``None`` to leave unchanged.
      :param note: New note, or ``None`` to leave unchanged.
      :param annotate: New annotator flag, or ``None`` to leave unchanged.

      :returns: The resulting :class:`ReviewRecord`.
